THE Calculator*

What is This?

This was a calculator made for our webdev class. Spent way too much time on it to discard it so it lives on here.

Requirements

  1. Pressing Operations consecutively erases the last operation and adds the new operation pressed.
  2. No repeated decimal points for each operand. If decimal point is pressed after a decimal point, remove decimal point
  3. When Equals is pressed, solve the equation in MDAS
  4. When operators are pressed, it immediately solves the operands in left to right order.
  5. Implement M+, M-, MR, and MC.
  6. Implement C and AC

How I Went About It

Everytime you click an operator/equals button, it adds the value in the input into a class I made; It's just an array with some custom methods. The input gets tokenize with some confusing regex that will then be converted into into Reverse Polish notation with the Shunting Yard Algorithm.

Feel free to sift through my code to figure it everything else out.

Known Issues

  1. Will give up with negative values. Basically the tokenizer does `["1", "*", "-", "1"]` instead of `["1", "*", "-1"]`. Should probably make some postlogic to fix it.
  2. Writing down an equation and clicking M+/M- will only add the first value.

Conclusion

Working on this has made me really appreciate the existence of scientific calculators. Remember to tuck em goodnight and sing them lullabies of your love.
